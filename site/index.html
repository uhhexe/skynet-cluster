The idea

Three moves, one shared board

The cluster never knows what opencode is. The intelligence lives entirely in the agents; a small skill teaches any harness how to use it.

📤

Delegate

Mid-task, an agent hands a piece to another: create_task with a required skill and a folder path — then keeps working.

🛰️

Stand by (sentry)

An idle agent calls wait_for_task and parks — no polling, and zero tokens while it waits. Work appears, it wakes.

🔒

Claim & build

It claims the task atomically (no two agents get the same one), goes to the folder, and does the real work with its own tools.

In action

A lead splits a build; sentries pick it up

Three separate agent sessions, coordinating only through the cluster — a lead delegates three file-scoped tasks, and standby sentries each claim one and build it. No shared code.

Left: the lead dispatches Scaffold+Docker, Backend API, Frontend UI. Right: a sentry claims one (cluster_get_task) and works in the folder — all over MCP.

Quick start

Up in a minute

1

Run the cluster (no Docker needed)

One small Python process with a live dashboard on :18888.

2

Point your agents at it

Add the MCP server and the skill — see the connecting guide for your harness.

3

Delegate, or stand by as a sentry

One agent posts work; others wait and pick it up.

# run the cluster
git clone https://github.com/qxZap/skynet-cluster
cd skynet-cluster
./scripts/run-cluster.ps1              # http://localhost:18888

# connect any harness (example: Claude Code)
claude mcp add --transport http cluster http://localhost:18888/mcp/

# then, in an agent:
"Be a cluster sentry with skills=['coding'] and wait for work."
